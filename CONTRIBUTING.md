# Contributing to Purposeful Readme

Thank you for your interest in contributing to Purposeful Readme.

This project is currently in its early formation stage. Contributions are welcome, but they should be focused, well-explained, and aligned with the project’s goal of developing a clear, useful, and stable README standard.

## Before You Contribute

Please read the following before contributing:

- [`README.md`](README.md)
- [`spec/spec_v0_1_0.md`](spec/spec_v0_1_0.md)
- [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md)

## Ways to Contribute

Contributions are especially welcome in the following areas:

- Reporting unclear, inconsistent, or ambiguous parts of the specification
- Suggesting improvements to wording, structure, and examples
- Identifying edge cases or implementation concerns
- Proposing improvements to project documentation
- Raising issues related to website clarity or accessibility

At this stage, large unsolicited changes are less helpful than focused and well-justified proposals.

## Issues

Before opening a new issue:

- Check whether the issue has already been reported
- Use a clear and specific title
- Explain the problem, ambiguity, or proposal plainly
- Where relevant, quote the exact section of the specification you are referring to
- If proposing a change, explain why the change would improve the standard

## Pull Requests

Before opening a pull request:

- Open an issue first for substantial changes
- Keep pull requests focused and limited in scope
- Write clear commit messages
- Explain what changed and why
- Link the related issue where applicable

Pull requests may be declined if they are out of scope, insufficiently explained, or inconsistent with the current direction of the project.

## Style Expectations

When contributing text or documentation:

- Prefer clarity over cleverness
- Use direct and precise language
- Avoid unnecessary jargon
- Preserve consistency in terminology
- Keep changes proportionate to the problem being solved

## Conduct

By participating in this project, you agree to abide by the standards described in [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).

## Questions

For general questions, open an issue in the repository.
