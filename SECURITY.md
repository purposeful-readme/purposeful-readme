# Security Policy

## Supported Versions

At this stage, Purposeful Readme is an early-stage documentation and standards project.

Security support is currently provided for the latest development state of the repository at:

- [`purposeful-readme/purposeful-readme`](https://github.com/purposeful-readme/purposeful-readme)

Older snapshots, drafts, and superseded versions may not receive security updates.

## Reporting a Vulnerability

If you believe you have found a security vulnerability related to this project, please report it privately and do not disclose it publicly at first.

Examples may include:

- A vulnerability in the project or its deployment configuration
- A security issue in repository automation or project infrastructure
- Exposure of secrets, tokens, credentials, or sensitive configuration
- A supply-chain concern affecting project dependencies or build/deployment workflows

Please send reports to: security[at]purposeful-readme.org

Please include, where possible:

- A clear description of the issue
- Steps to reproduce
- The affected repository, file, feature, or deployment component
- Any proof of concept, screenshots, or logs that help explain the issue
- Your assessment of possible impact

## What to Expect

The maintainer(s) will aim to:

- Acknowledge receipt of the report within a reasonable time
- Investigate the issue privately
- Confirm whether the report is valid
- Take proportionate remediation steps where needed
- Credit the reporter where appropriate and desired

## Scope Note

The Purposeful Readme specification itself is primarily a documentation standard and may not, by itself, create software vulnerabilities in the same way an executable application can. However, the repository, automation, and related infrastructure may still present security-relevant issues and should be reported responsibly.

## Disclosure

Please do not open public GitHub issues for suspected security vulnerabilities before the issue has been reviewed.

Coordinated disclosure is preferred.
